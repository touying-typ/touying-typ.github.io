# alternatives

<Signature>
  <code><SignatureName>alternatives</SignatureName>(<SignatureParam>[self: none](#self), </SignatureParam><SignatureParam>[start: 1](#start), </SignatureParam><SignatureParam>[repeat-last: true](#repeat-last), </SignatureParam><SignatureParam>[..args](#args)</SignatureParam>)<SourceLink href="https://github.com/touying-typ/touying/blob/main/src/utils.typ#L1368" /></code>
</Signature>

`#alternatives` is able to show contents sequentially in subslides.

Example: `#alternatives[Ann][Bob][Christopher]` will show "Ann" in the first subslide, "Bob" in the second subslide, and "Christopher" in the third subslide.

<Parameters>

#### <ParamName>start</ParamName> : `int` <Default>`1`</Default> {#start}

<Param>

The starting subslide number. Default is `1`.

</Param>

#### <ParamName>repeat-last</ParamName> : `bool` <Default>`true`</Default> {#repeat-last}

<Param>

Whether the last alternative should persist on all remaining subslides. Default is `true`.

</Param>

#### <ParamName>position</ParamName> : `alignment` {#position}

<Param>

The alignment of alternatives within the reserved space. Default is `bottom + left`.

</Param>

#### <ParamName>stretch</ParamName> : `bool` {#stretch}

<Param>

Whether to stretch all alternatives to the maximum width and height. Default is `false`. Important: If you use a zero-length content like a context expression, you should set `stretch: false`.

</Param>

</Parameters>

**Returns:** `content`
