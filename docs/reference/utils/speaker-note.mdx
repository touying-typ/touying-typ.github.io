# speaker-note

<Signature>
  <code><SignatureName>speaker-note</SignatureName>(<SignatureParam>[self: none](#self), </SignatureParam><SignatureParam>[mode: "typ"](#mode), </SignatureParam><SignatureParam>[setting: it => it](#setting), </SignatureParam><SignatureParam>[subslide: none](#subslide), </SignatureParam><SignatureParam>[note](#note)</SignatureParam>)<SourceLink href="https://github.com/touying-typ/touying/blob/main/src/utils.typ#L1557" /></code>
</Signature>

Speaker notes are a way to add additional information to your slides that is not visible to the audience. This can be useful for providing additional context or reminders to yourself.

Multiple calls on the same slide are combined (accumulated), so all notes are shown together.

Example: `#speaker-note[This is a speaker note]`

<Parameters>

#### <ParamName>self</ParamName> : `dictionary` <Default>`none`</Default> {#self}

<Param>

The current presentation context.

</Param>

#### <ParamName>mode</ParamName> : `str` <Default>`"typ"`</Default> {#mode}

<Param>

The mode of the markup text, either `typ` or `md`. Default is `typ`.

</Param>

#### <ParamName>setting</ParamName> : `function` <Default>`it => it`</Default> {#setting}

<Param>

A function that takes the note as input and returns a processed note.

</Param>

#### <ParamName>subslide</ParamName> : `none` | `auto` | `int` | `array` | `str` <Default>`none`</Default> {#subslide}

<Param>

Restricts the note to specific subslides, similar to `only`. - `none`: shown on all subslides. - `auto`: automatically determined from the current pause position (default when called via `#speaker-note`). - int, array, or string: shown only on the specified subslides.

</Param>

#### <ParamName>note</ParamName> : `content` {#note}

<Param>

The content of the speaker note.

</Param>

</Parameters>

**Returns:** `content`
